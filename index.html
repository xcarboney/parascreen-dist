Frequently asked

Answers before you ask.

Does ParaScreen work on Intel Macs?

No — Apple Silicon (M1 or later) only. ScreenCaptureKit and Vision's on-device OCR depend on the Neural Engine for real-time performance, so Intel support isn't planned.

Is my data ever uploaded?

Never. OCR and scrolling-screenshot stitching both run entirely on-device using Apple's Vision framework. There's no network code in the capture path. Everything stays on your Mac.

How does scrolling capture work?

We grab a frame every 180ms while you scroll, then use VNTranslationalImageRegistrationRequest to compute the exact pixel offset between consecutive frames. The non-overlapping rows get stitched onto a running canvas. Scrolling backwards crops the stitch shorter.

Can I record system audio?

Yes. ScreenCaptureKit captures system audio without a kernel extension or a virtual audio driver. Toggle it on in the recording setup panel.

Where are my captures saved?

By default, ~/Library/Application Support/ParaScreen/captures/. You can change the folder in Settings → General. All captures are also indexed in a browsable library.

Can I remap the hotkeys?

Every action. Any key. Single function-key bindings allowed. Conflicts auto-resolve by swapping. See Settings → Shortcuts.

Will you ship ParaScreen for iOS / iPad?

Not planned. The scrolling-screenshot feature is built-in on iOS, and ScreenCaptureKit is macOS-only. ParaScreen is a Mac-first app.
